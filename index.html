§ I · THE STORY

탄생 배경

hstack은 처음부터 설계된 프레임워크가 아니다. 한 번의 실패, 한 번의 성공 세션, 그리고 그 세션을 역추적한 결과물이다.

  1. 01
    촉발 · The Trigger

    두 개의 정부 과제, 촉박한 일정

    팀이 두 개의 정부 과제 제안서를 매우 촉박한 일정 안에 동시에 작성해야 하는 상황이 발생했다. 수동으로 감당하기 어려운 분량이었다.

  2. 02
    첫 시도, 그리고 실패 · The Failure

    "skills를 조합한 agent를 만들자" — 작동하지 않았다

    처음에는 generic한 접근을 시도했다. 여러 skill을 pipeline으로 연결한 agent를 구현했지만, 출력이 기대에 미치지 못했다. 구조는 있었으나 판단이 없었다.

  3. 03
    전환점 · The Turn

    형진님이 직접 prompting을 맡다

    어느 Claude Code 세션에서 형진님이 prompting을 직접 맡았다. 세션 시작부터 초안 완성까지, 그가 대화를 이끌었다. 그리고 실제로 쓸 수 있는 초안이 나왔다.

  4. 04
    관찰 · The Observation

    그 세션을 역추적하다

    무엇이 달랐는가 — 어떤 순서로 질문을 던졌는가, 어떤 판단 기준을 사용했는가, 어떤 충돌을 어떻게 해결했는가. 그 안에는 반복되는 사고 패턴이 있었다.

  5. 05
    형식화 · The Codification

    12단계 + 14개 규칙

    그렇게 드러난 형진님의 의사결정·검증·피드백 처리 방식을 12단계 + 14개 규칙으로 구조화해 Claude Code 스킬로 변환했다. 이름의 "h"는 형진님, "stack"은 그의 사고 스택. (명명 규칙은 YC CEO Garry Tan의 gstack을 따랐다.)

§ VI · INSTALLATION

시작
하기

4개의 절차. 5분 안에 새 프로젝트에서 hstack을 사용할 수 있다.

  1. I.

    스킬 복사

    hstack 디렉토리를 Claude Code 스킬 폴더에 복사한다.

    $ cp -r hstack ~/.claude/skills/
  2. II.

    프로젝트 초기화

    프로젝트 폴더에 input/ 구조를 만든 뒤 원천 자료 배치.

    $ mkdir -p input/{rfp,template,company_profile,meeting,reference,sample_data}
$ mkdir -p converted working research output
  3. III.

    Claude Code 시작

    해당 폴더에서 Claude Code 실행 후 스킬 트리거.

    $ claude
> input/ 폴더의 RFP로 제안서 작성해줘
  4. IV.

    12단계 따라가기

    각 단계 완료 기준에 도달하면 확인 후 다음으로.

    > 다음 단계로 진행해주세요  (또는 피드백 제공)

형진님의 제안서 작성 사고 방식 · Claude Code Skill. 암묵지가 스킬이 된다.
